最近做了一个电商数据导入与清洗的实验，全程基于Hadoop + Hive环境操作，踩了一些坑也积累了不少实用经验，整理成博客记录下来，方便自己后续回顾，也希望能帮到有需要的小伙伴～

## 一、实验背景

在电商数据分析的完整流程中，数据导入是将原始数据接入分析环境的第一步，也是基础步骤；而数据清洗则是提升数据质量、避免后续分析结果失真的关键环节。本次实验基于一份5.6G的电商CSV数据集，完整完成了「本地数据→HDFS→Hive」的导入流程，同时针对性解决了数据中的缺失值、重复值等问题，全程实操性拉满。

## 二、实验目的

- 熟练掌握本地电商数据导入Hadoop + Hive环境的完整流程，搞定环境配置与数据上传的核心操作；

- 学会用Hive SQL处理数据清洗中的常见问题（缺失值、重复值），提升数据质量；

- 积累实操经验，为后续深入做电商数据分析（比如用户行为、商品趋势分析）打基础。

## 三、实验环境配置

环境配置是本次实验最耗时的部分，踩了不少版本兼容、路径配置的坑，整理了最简洁的实操步骤，避免大家走弯路。

### （一）Hive配置

1. **下载与解压**：从Apache Hive官网下载3.1.2版本，解压到/usr/local目录并改名hive，方便后续操作。
```bash
# 下载安装包
wget https://mirrors.huaweicloud.com/apache/hive/hive-3.1.2/apache-hive-3.1.2-bin.tar.gz
# 解压到指定目录
sudo tar -zxvf apache-hive-3.1.2-bin.tar.gz -C /usr/local
# 重命名
sudo mv /usr/local/apache-hive-3.1.2-bin /usr/local/hive`
```

2. **配置环境变量**：编辑/etc/profile文件，添加Hive环境变量，保存后生效。
```bash
sudo vim /etc/profile
# 末尾添加以下内容
export HIVE_HOME=/usr/local/hive
export PATH=$PATH:$HIVE_HOME/bin
# 生效环境变量
source /etc/profile`
```

3. **元数据存储配置（MySQL）**：用MySQL存储Hive元数据，需安装MySQL、创建元数据库、配置JDBC驱动。
```bash
# 安装MySQL
sudo apt install mysql-server
# 登录MySQL，创建元数据库
mysql -u root -p
CREATE DATABASE hivemetastore;
# 下载JDBC驱动并复制到Hive的lib目录
sudo wget https://repo1.maven.org/maven2/org/mariadb/jdbc/mariadb-java-client/3.2.0/mariadb-java-client-3.2.0.jar
sudo cp mariadb-java-client-3.2.0.jar /usr/local/hive/lib`
```
4. **修改Hive配置文件**：在/usr/local/hive/conf目录下，基于模板创建hive-site.xml，配置MySQL连接信息，同时解决Hadoop与Hive的guava版本差异。
```bash
# 解决guava版本差异
sudo rm $HIVE_HOME/lib/guava-19.0.jar
sudo cp $HADOOP_HOME/share/hadoop/common/lib/guava-27.0-jre.jar $HIVE_HOME/lib/
# 复制并编辑hive-site.xml
cd /usr/local/hive/conf
sudo cp hive-default.xml.template hive-site.xml
sudo vim hive-site.xml
```

在hive-site.xml中添加以下核心配置（替换自己的MySQL用户名和密码）：
```textplain
javax.jdo.option.ConnectionURL jdbc:mysql://localhost:3306/hivemetastore?createDatabaseIfNotExist=true&amp;useSSL=false&amp;allowPublicKeyRetrieval=true
javax.jdo.option.ConnectionDriverName com.mysql.cj.jdbc.Driver
javax.jdo.option.ConnectionUserName user
javax.jdo.option.ConnectionPassword 123456LJJ
hive.metastore.schema.verification false
```

5. **初始化元数据库与启动服务**：初始化Hive元数据库，重启Hadoop和Hive服务，确保配置生效。
```bash
# 初始化元数据库
schematool -dbType mysql -initSchema --verbose
# 重启Hadoop和Hive服务
$HADOOP_HOME/sbin/stop-all.sh
$HADOOP_HOME/sbin/start-all.sh
$HIVE_HOME/bin/hiveserver2 start`
```

### （二）Hive与HDFS关联配置

Hive需要访问HDFS存储数据，需创建HDFS目录并配置权限，同步修改Hive配置。

```bash
# 创建HDFS目录并设置权限
hdfs dfs -mkdir -p /user/hive/warehouse
hdfs dfs -mkdir /tmp
hdfs dfs -chmod g+w /user/hive/warehouse
hdfs dfs -chmod g+w /tmp
```

添加HDFS配置（修改hive-site.xml）
```xml
fs.defaultFS hdfs://namenode:9000
hive.exec.scratchdir /tmp/hive
hive.downloaded.resources.dir /tmp/hive/resources
```

### （三）Tez配置（优化Hive执行效率）

为了提升Hive查询效率，配置Tez作为执行引擎，步骤如下：

```bash
# 下载并解压Tez
wget https://mirrors.tuna.tsinghua.edu.cn/apache/tez/0.10.2/apache-tez-0.10.2-bin.tar.gz
tar -zxvf apache-tez-0.10.2-bin.tar.gz -C /opt/tez
# 上传Tez到HDFS
hdfs dfs -mkdir -p /apps/tez
hdfs dfs -put /opt/tez/* /apps/tez
```

配置Hive使用Tez（修改hive-site.xml）
```xml
hive.execution.engine tez
hive.tez.container.size 2048
hive.tez.java.opts -Xmx1536m
```

### （四）环境测试

创建测试表并插入数据，验证Hive与Hadoop关联成功：

```sql
CREATE TABLE test_table (id INT, name STRING);
INSERT INTO TABLE test_table VALUES (1, 'test');
```

能正常查询到数据，说明环境配置无误～

## 四、实验方法与步骤（核心实操）

### （一）数据导入

1. **本地数据准备**：确认本地存储的电商数据集（CSV格式，5.6G）完整无损坏，路径为`/home/user/UserBehavior.csv`。

2. **上传至HDFS**：使用Hadoop命令将本地数据上传到Hive对应的仓库目录，确保路径正确。
```bash
hdfs dfs -put /home/user/UserBehavior.csv /user/hive/warehouse/taobao_consume.db`
```

3. **创建Hive表**：根据数据集字段结构，创建对应的Hive表，选用ORC格式提升查询性能。
```sql
CREATE TABLE ecommerce_user_purchase (
    用户ID STRING,
    商品ID STRING,
    购买时间 TIMESTAMP,
    购买数量 INT,
    支付金额 DOUBLE
) STORED AS ORC;`
```

4. **数据加载**：将HDFS上的CSV数据加载到Hive表中，由于数据量较大，需耐心等待执行完成。
```sql
LOAD DATA INPATH '/user/hive/warehouse/taobao_consume.db/UserBehavior.csv' INTO TABLE ecommerce_user_purchase;
```

### （二）数据清洗

数据清洗的核心是解决缺失值、重复值问题，本次用Hive SQL完成，步骤清晰且高效。

1. **缺失值处理**：
        

    - 先用COUNT() + IS NULL统计缺失值：「购买时间」缺失率0.1%，「支付金额」缺失率5%；

    - 「购买时间」：用向前填充法（窗口函数），填充该用户上一次购买时间；

    - 「支付金额」：因对后续分析影响较大，直接删除含缺失值的记录，共删除约4.8万条。

2. **重复值处理**：用DISTINCT关键字对全表去重，共删除约1万条重复记录，确保数据唯一性。
```sql
-- 去重查询
SELECT DISTINCT * FROM ecommerce_user_purchase;`
```

## 五、实验结果

- **数据导入**：成功将本地5.6G电商数据完整导入Hive，原始数据95万条，字段与原始数据集完全匹配；

- **数据清洗**：清洗后剩余90万条记录，缺失值、重复值问题全部解决，数据质量满足后续分析需求，可用于用户购买行为、商品销售趋势等深入分析。

## 六、实验踩坑与解决方案（重点！）

本次实验最棘手的问题：**数据上传至HDFS时，提示空间不足，导致Hadoop进入安全模式**。

解决方案：通过扩展服务器卷容量，释放足够存储空间后，执行命令退出安全模式，重新上传数据，最终成功完成导入。
